# Contributing

WatermelonDB is an open-source project and it needs your help to thrive!

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request:

1. **Check existing issues** - Make sure the issue hasn't been reported
2. **Create a new issue** - Provide as much detail as possible
3. **Include examples** - Code examples help us understand the problem

### Contributing Code

1. **Fork the repository** - https://github.com/Nozbe/WatermelonDB
2. **Create a branch** - `git checkout -b feature/my-feature`
3. **Make your changes** - Follow coding standards
4. **Write tests** - Make sure tests pass
5. **Submit a PR** - Include description of changes

### Good First Issues

If you're new to WatermelonDB, check out [good first issues](https://github.com/Nozbe/WatermelonDB/labels/good%20first%20issue).

## Development Setup

### Prerequisites

- Node.js
- React Native development environment
- Xcode (for iOS) or Android Studio (for Android)

### Setup

```bash
# Clone the repository
git clone https://github.com/Nozbe/WatermelonDB.git
cd WatermelonDB

# Install dependencies
yarn install

# Run tests
yarn test
```

## Coding Standards

- **Follow existing code style** - Use same formatting and patterns
- **Write tests** - All features should have tests
- **Document code** - Add JSDoc comments for public APIs
- **Keep commits focused** - One feature/fix per commit

## Testing

```bash
# Run all tests
yarn test

# Run specific test file
yarn test path/to/test.js

# Watch mode
yarn test --watch
```

## Submitting Pull Requests

When submitting a PR:

1. **Write clear description** - Explain what and why
2. **Link related issues** - Reference any related issues
3. **Add tests** - Include tests for new features
4. **Update documentation** - Update docs if needed
5. **Ensure tests pass** - All CI checks must pass
